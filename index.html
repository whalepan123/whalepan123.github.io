ManiBeat

Wish · Frequency · Music
角色独立孵化 阶段已上架 · iOS / visionOS 核心机制愿望绑定的每日仪式

把年轻人的「显化 manifestation」情绪需求与 AI 音乐生成结合：写下你的意图，选择疗愈频率（432Hz / 528Hz / Theta…）、时长与风格（Ambient / Lo-Fi / Orchestral），AI 生成围绕这个愿望的专属音轨——把泛化的 AI 音乐落成每日显化仪式。

关键决策：不做泛化的 AI 音乐生成器——把每次生成绑定在「一个愿望」上。同一意图每天常听常新，novelty 变成仪式，仪式变成留存。

AI 工程：生成走固定参数契约（频率 / 时长 / 风格），实现可控随机——同一愿望、不同音轨、一致气质。

AI 生成音乐 + 显化意图 订阅制 Premium iOS · visionOS 上架
项目故事：显化潮流 × AI 音乐的空档

背景：「显化」是 Z 世代最大的情绪自助潮流之一，但市面上的显化产品大多停留在文字肯定语与日记；另一边，AI 音乐生成能力刚成熟，却多是「玩一次就走」的 novelty。两者之间存在空档——把 AI 音乐变成围绕一个愿望的每日聆听仪式。

Polaris

Daily Feed × Explore
角色独立 Owner · 在职 0→1 阶段孵化中 · 概念已脱敏 核心亮点双引擎 + PGC 闭环

核心命题：学习工具的留存困境，在于「今天该学什么」永远被丢回给用户。Polaris 让产品接管这个问题——今日 Feed 用遗忘曲线与原题拆解生成每日计划，每张卡片都写明「为什么是它」；探索把任何主题拆成关卡路径，像 Blinkist 拆书一样拆解一切要学的东西。

关键决策：自动化接管之前先赢得信任——每张推荐卡都暴露理由（掌握度滑落 / 上周你在变式题栽过）。先讲理由，再谈托管。

AI 工程：多模型按成本分工——抽取 / 大纲用轻模型，面向学习者的编排生成用强模型；输出用 schema 强约束；掌握度用 SM-2 稳定性 + 遗忘曲线建模，只在真实作答后更新。

最关键的一环：学完的备考路径可以一键发布成课程（PGC）——每个认真备过考的人，都是下一个考生最好的老师。这让「学」与「教」在同一个产品里闭环。

Duolingo 式关卡引擎 遗忘曲线驱动每日 Feed 路径 PGC 闭环
项目故事：为什么是双引擎

背景：工具型学习产品的共同困境是「用户传完资料、做完题就走」——缺少回答「今天该学什么」的主动性抓手。调研 8+ 竞品后发现两个事实：「任意主题→关卡路径」的 to C 产品至今没有赢家；而「学习路径的一键分享」几乎是空白。

我做了什么：以独立 Owner 在职推进——竞品结构化拆解 → 双引擎产品定义 → 关卡内容范式与题型设计（先猜 → 揭示 → 命名 → 用到你身上）→ 本演示的全部交互与前端由我实现。

方法上的坚持：游戏化只奖励真实学习行为（不做惩罚型机制）；先让人试、再让人懂；掌握度只在真实作答后更新，被动浏览不算学会。

* 在职项目的脱敏概念演示——命名、数据与视觉均为重构，不代表任何未发布产品的最终形态。

薛定谔的喵

Schrödinger's Meow · Observe Your Cat
角色个人全栈 · 概念到实现 阶段可交互原型 核心亮点观测隐喻 × 评估养成

一个帮年轻人与「不确定性」和解的心情疗愈 app。借薛定谔的猫的概念：领养一只住在盒子里的小猫作为你的观测对象——在你观测之前，它的一天什么都可能发生；每次发起对话都是一次观测，小猫会回你一张天马行空的照片证明它正在干什么，再用语聊陪你聊聊今天心里的天气、那些悬而未决的事。

关键决策：拒绝把它做成「冥想式功课」——用观测隐喻把与不确定共处拆成一次次轻交互（观测、照片、聊天），让自我关怀像喂猫，而不像治疗。

AI 工程：照片走图像生成、陪伴走 LLM 语聊；心理评估做成随对话逐轮累积的标签体系，而非一次性问卷。

对话在持续沉淀你的心理评估标签与「不确定性耐受度」评分；配合每日观测、记录「没炸的事」、晚安信等游戏化任务，让心理状态成为可以温柔养成的东西。

观测 = 核心交互隐喻 AI 图像 + 语聊陪伴 心理评估标签体系
项目故事：为什么是薛定谔的猫

背景：年轻人的焦虑大多来自对不确定的灾难化想象——offer 没回、体检报告没出、消息未读。市面的冥想 / 正念 app 太「正确」、太像功课；缺一个轻盈的、有陪伴感的入口，把「与不确定共处」变成日常练习而不是治疗。

产品机制：薛定谔的猫是完美的隐喻载体——「盒子未开，结局未定」。用户上传或从库里选一只小猫建档，观测（对话）时 AI 生成小猫的奇想照片作为「观测证据」，语聊围绕不确定性话题展开；每轮对话更新心理评估标签与耐受度评分，游戏化任务驱动每日回访与评分成长。

状态：个人独立开发作品——从概念、交互到实现均由本人完成，用于探索「AI 陪伴 × 心理自助」的产品范式。

Agent 增长引擎

Signal → Creative → Product → Feedback
角色独立搭建 阶段已上线验证 核心结果第一大投放来源 · 获新 +100%

素材获客的老毛病是迭代慢、靠灵感。我独立搭建的多 Agent 管线把它变成一条自动闭环流水线：信号 Agent 持续扫描平台趋势与用户讨论；创意 Agent 把信号翻译成素材脚本并预估表现；承接 Agent 让 App 首屏与深链跟着素材同步变化；反馈 Agent 读投放数据、定位弱点，自动发起下一轮迭代——人只在关键节点做判断。

关键决策：反馈不达标时，诊断自动回流创意层重跑——而不是回到人的待办清单。只有真闭环才能压缩迭代周期。

AI 工程：每个 Agent 用固定 JSON 契约串联管线；反馈 Agent 的判定阈值由历史投放数据校准。

同一套框架延伸出三个研究 Agent（Idea 孵化 / 场景痛点 / 爆款素材拆解），支撑新品孵化：最终验证 1 个新产品方向，并把洞察落成 3 个可上线功能。

第一大投放消耗来源 获新效率 +100% 3 类研究 Agent
项目故事：为什么要把 GTM 做成 Agent

背景：投放素材的生产是典型的「灵感瓶颈」——想法靠人憋、验证靠等数据、素材和产品承接两张皮。新产品孵化期尤其伤：方向探索慢、验证成本高。

架构关键：把「热点 → 创意 → 承接 → 反馈」四个环节各自 Agent 化，用数据接口串成闭环；反馈不达标时自动带着诊断回流创意层重跑。

结果：系统上线验证后成为第一大投放消耗来源，获新效率相对提升 100%+，并贡献付费留存增量。
